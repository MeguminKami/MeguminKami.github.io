Simulation Framework

A high-performance simulation framework for testing stochastic methods in network clock synchronization, developed for distributed systems research at INESCTEC.

Research C Distributed Systems Simulation
View on GitHub Read Paper

Overview

This simulation framework enables researchers to test and validate stochastic clock synchronization algorithms in distributed network environments. Built for performance and extensibility, it supports large-scale experiments with real-time monitoring capabilities.

System Architecture

Distributed node architecture with stochastic synchronization

Problem

Network clock synchronization is critical for distributed systems, but testing synchronization algorithms in real networks is costly and difficult to reproduce. Researchers need a controlled environment to evaluate stochastic methods under various network conditions and failure scenarios.

Solution

I developed a C-based simulation framework that:

  • Models distributed network nodes with configurable clock drift
  • Implements multiple stochastic synchronization algorithms
  • Simulates network delays, packet loss, and Byzantine failures
  • Provides real-time performance metrics and visualization
  • Supports large-scale experiments (1000+ nodes)

Technical Stack

C Makefiles CMake Git Python (analysis) gnuplot (visualization)

Key Features

Stochastic Methods

Implements probabilistic approaches to clock synchronization, allowing nodes to achieve consensus despite unreliable communication channels and Byzantine actors.

Performance Monitoring

Real-time metrics track synchronization accuracy, convergence time, message overhead, and fault tolerance. Data exports enable detailed statistical analysis.

Extensible Architecture

Modular design allows researchers to plug in new synchronization algorithms, network models, and failure scenarios without modifying core framework code.

Results

  • Enabled testing of novel synchronization algorithms at scale
  • Reduced research iteration time through reproducible experiments
  • Published findings in distributed systems research community
  • Provided foundation for follow-up hardware implementations
